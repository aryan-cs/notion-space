\begin{abstract}
\noindent The Notion-Based Reasoning System (\NBRS) performs reasoning by traversing a typed directed acyclic graph of operations and resolving each operation node with a shared, capacity-limited neural processor invoked only when its inputs are available. This document develops the formal theory of the architecture. We formalise the executor as a deterministic state machine over a typed DAG and establish its core operational properties: argument availability is preserved as an invariant, the resulting computation is deterministic, order-independent across topological schedules, terminates in a number of node invocations linear in graph size, and reduces to oracle evaluation under a primitive-accurate processor. We contrast its inductive bias with a weight-tied graph transformer carrying a causal ancestor mask and with a sequence transformer over a serialised DAG, and prove an expressivity containment between the three. The executor's per-step input is bounded by the maximum primitive arity independently of graph depth; combined with a stochastic processor of per-invocation error $\eps$, this yields a capacity--depth decoupling: depth-$d$ graphs are resolved correctly with probability at least $(1-\eps)^d$ and per-step parameter count is constant in $d$. The processor sees only the local context of each node, which gives the architecture a clean local-context invariance principle and, under marginal-invariance of the local context across depths, a depth-independent expected per-node error. We extend the framework to autonomous graph construction as a Markov decision process with deterministic transitions and a polynomially-bounded action space, and we state the formal questions associated with extending the primitive library through subgraph abstraction.
\end{abstract}

\section{Introduction}
\label{sec:intro}

A neural network that simulates multi-step reasoning through autoregressive token prediction must perform two distinct functions simultaneously: deciding what sub-problems to solve, and solving them. The two functions share parameters, share context, and share representational bandwidth. As compositional depth grows, the constraints they place on each other tighten. This is the structural reason behind the observed degradation of token-level reasoning with depth: a fixed-capacity sequence model cannot allocate independent computational resources to each step of a multi-step computation.

The \NBRS\ separates the two functions architecturally. Reasoning is represented as a typed directed acyclic graph of \emph{notions}---operations on typed values---with edges denoting argument dependencies. Execution is dependency-gated: a node fires only when all of its arguments are resolved, and at that point a single, shared, capacity-limited neural module (the \emph{node processor}) is invoked on the local context of the node. The same parameters are reused at every node, unrolled in time rather than in width. Compositional depth grows the number of node invocations; it does not consume neural parameters, attention positions, or representational bandwidth.

This document develops the theory. We define the typed DAG and its operational semantics, prove the executor's invariants, derive its information-theoretic properties, and characterise the conditions under which depth-generalisation holds. We then formulate the natural extensions: autonomous graph construction by a learned policy, and extension of the primitive library through subgraph abstraction. Throughout, the goal is to produce statements that admit precise analysis and clean comparison with sequence-based and graph-based alternatives.

\section{Formal Preliminaries}
\label{sec:prelim}

We work in a typed first-order setting. Values have types; operations have input and output types; reasoning is a typed computation organised as a DAG.

\subsection{Types and values}

\begin{definition}[Type system]
\label{def:types}
A type system is a pair $(\Types, \Val)$ where $\Types$ is a finite, decidable set of types and $\Val = \bigsqcup_{\tau \in \Types} \Val_\tau$ is a disjoint union of value domains. We write $v : \tau$ for $v \in \Val_\tau$.
\end{definition}

\subsection{Primitive operations}

\begin{definition}[Primitive operation]
\label{def:primitive}
A primitive operation is a tuple
\[
\tau \;=\; \bigl(\mathsf{name}_\tau,\; \mathsf{in}_\tau,\; \mathsf{out}_\tau,\; f_\tau\bigr),
\]
where $\mathsf{in}_\tau \in \Types^\ast$ is a (possibly empty) tuple of input types of length $k_\tau$, $\mathsf{out}_\tau \in \Types$ is the output type, and $f_\tau$ is a total function
\[
f_\tau \;:\; \Val_{\tau_1} \times \cdots \times \Val_{\tau_{k_\tau}} \;\to\; \Val_{\mathsf{out}_\tau}.
\]
We refer to $f_\tau$ as the \emph{semantic function} of $\tau$; it defines the mathematical meaning of the primitive, independently of any neural realisation.
\end{definition}

\begin{definition}[Primitive library]
\label{def:library}
A primitive library is a finite, ordered set $\Lib = \{\tau_1, \dots, \tau_K\}$ of primitives. The library is type-closed if for every $\tau \in \Lib$ and every $\sigma \in \mathsf{in}_\tau$ either $\sigma$ is an input type of the overall task or some primitive in $\Lib$ has output type $\sigma$.
\end{definition}

\subsection{Notions and notion DAGs}

\begin{definition}[Notion]
\label{def:notion}
A notion over a library $\Lib$ is a tuple
\[
n \;=\; \bigl(\mathsf{id}_n,\; \tau_n,\; \Spec_n,\; \Parents_n\bigr),
\]
where $\mathsf{id}_n$ is a unique identifier, $\tau_n \in \Lib$, $\Spec_n$ encodes any auxiliary parameters of the primitive (constants, shift offsets, comparison directions), and $\Parents_n = (p_1, \dots, p_{k_{\tau_n}})$ is the ordered tuple of parent identifiers, one per input slot of $\tau_n$. A notion with $\Parents_n = ()$ is a leaf.
\end{definition}

\begin{definition}[Notion DAG]
\label{def:notion-dag}
A notion DAG over $\Lib$ is a pair $G = (V, r)$ where $V$ is a finite set of notions, every parent identifier in $\bigcup_{n \in V} \Parents_n$ refers to a notion in $V$, the induced parent relation is acyclic, and $r \in V$ is a distinguished output notion.
\end{definition}

\begin{definition}[Well-formedness]
\label{def:wellformed}
A notion DAG $G = (V, r)$ is well-formed if (i) the parent relation is acyclic; (ii) every parent identifier in $\Parents_n$ refers to a notion in $V$; (iii) types match along every edge: for each $n \in V$ and each $i \in \{1, \dots, k_{\tau_n}\}$, the $i$-th parent $p_i$ satisfies $\mathsf{out}_{\tau_{p_i}}$ equal to the $i$-th declared input type of $\tau_n$; (iv) every non-leaf notion has exactly $k_{\tau_n}$ parents.
\end{definition}

All notion DAGs in the remainder of this document are assumed well-formed.

\begin{definition}[Compositional depth]
\label{def:depth}
The depth of a notion $n$ in a notion DAG $G$ is
\[
\dep(n) \;=\;
\begin{cases}
0 & \text{if } \Parents_n = (), \\
1 + \max_{p \in \Parents_n} \dep(p) & \text{otherwise.}
\end{cases}
\]
The depth of $G$ is $\dep(G) := \dep(r)$.
\end{definition}

A notion DAG generalises an abstract syntax tree by permitting value sharing: a single resolved value may feed multiple downstream operations. The remainder of this document handles the general DAG case.

\subsection{Reasoning tasks and oracle execution}

\begin{definition}[Reasoning task]
\label{def:reasoning-task}
A reasoning task over $\Lib$ is a pair $(G, \sigma)$ where $G = (V, r)$ is a well-formed notion DAG and $\sigma : V_{\mathrm{leaf}} \to \Val$ is a type-respecting assignment of values to the leaves of $G$. The ground-truth answer of the task is the value at the root produced by recursive substitution of the primitive semantic functions.
\end{definition}

Recursive substitution evaluates each non-leaf notion $n$ as
\[
\hat{v}_n \;=\; f_{\tau_n}\!\bigl(\hat{v}_{p_1}, \dots, \hat{v}_{p_{k_{\tau_n}}}\bigr),
\]
where $(p_1, \dots, p_{k_{\tau_n}}) = \Parents_n$. We record the order-independence of this evaluation for later use.

\begin{lemma}[Order independence of oracle execution]
\label{lem:oracle-order}
Let $G = (V, r)$ be a well-formed notion DAG with leaf assignment $\sigma$ and primitive semantic functions $\{f_\tau\}_{\tau \in \Lib}$. For any two topological orderings $\pi_1, \pi_2$ of $V$, the value $\hat{v}_n$ produced at each $n \in V$ by oracle evaluation is the same under $\pi_1$ and $\pi_2$.
\end{lemma}

\begin{proof}
Induction on $\dep(n)$. For $\dep(n) = 0$ we have $\hat{v}_n = \sigma(n)$, independent of order. For $\dep(n) > 0$, $\hat{v}_n$ depends only on $f_{\tau_n}$ and the parent values, each of which is order-invariant by the inductive hypothesis. Determinism of $f_{\tau_n}$ yields the conclusion.
\end{proof}

\section{The Graph-Structured Recurrent Executor}
\label{sec:gsre}

The Graph-Structured Recurrent Executor (\GSRE) is a deterministic state machine over a notion DAG that delegates value resolution at each node to a shared neural processor. This section defines the executor formally and establishes its core operational properties.

\subsection{Node states}

\begin{definition}[Node state]
\label{def:node-state}
During execution, every node $n \in V$ is in one of five states drawn from $\mathcal{S} = \{\PENDING, \READY, \EXEC, \RESOLVED, \FAILED\}$, written $\NState(n) \in \mathcal{S}$. Once $\NState(n) = \RESOLVED$, the associated value $v_n \in \Val_{\mathsf{out}_{\tau_n}}$ is fixed for the remainder of execution.
\end{definition}

The gating of $\PENDING \to \READY$ on parent state, depicted in \Cref{fig:state-machine}, is the architectural source of the executor's inductive bias: dependency order is not a learned soft attention pattern, it is a hard transition rule.

\subsection{The traversal engine}

The traversal engine drives the state machine. At each step it identifies all ready nodes, calls the shared processor on each, and updates state.

\begin{algorithm}[t]
\caption{The Graph-Structured Recurrent Executor}
\label{alg:gsre}
\begin{algorithmic}[1]
\Require Well-formed notion DAG $G = (V, r)$ with leaf assignment $\sigma$; processor $\NodeProc_\theta$.
\Ensure A (possibly partial) value assignment $v : V \rightharpoonup \Val$.
\State \textbf{for each} $n \in V$: $\NState(n) \gets \PENDING$
\State \textbf{for each} leaf $n \in V$: $v_n \gets \sigma(n)$;\; $\NState(n) \gets \RESOLVED$
\Loop
  \State $R \gets \bigl\{n \in V : \NState(n) = \PENDING \text{ and all } p \in \Parents_n \text{ have } \NState(p) = \RESOLVED\bigr\}$
  \State \textbf{for each} $n \in R$: $\NState(n) \gets \READY$
  \If{$R = \emptyset$}
    \If{$\NState(r) = \RESOLVED$} \State \Return $v$
    \ElsIf{$\NState(r) = \FAILED$ \textbf{or} no $\PENDING$ node has all-$\RESOLVED$ parents} \State \Return $v$
    \EndIf
  \EndIf
  \State $\mathcal{B} \gets R$
  \State \textbf{for each} $n \in \mathcal{B}$: $\NState(n) \gets \EXEC$
  \State $\hat{v}_\mathcal{B} \gets \NodeProc_\theta\bigl(\{(\tau_n, \Spec_n, (v_p)_{p \in \Parents_n}) : n \in \mathcal{B}\}\bigr)$
  \For{$n \in \mathcal{B}$}
    \If{$\hat{v}_{\mathcal{B},n} \ne \bot$}
      \State $v_n \gets \hat{v}_{\mathcal{B},n}$;\; $\NState(n) \gets \RESOLVED$
    \Else
      \State $\NState(n) \gets \FAILED$
    \EndIf
  \EndFor
\EndLoop
\end{algorithmic}
\end{algorithm}

\Cref{alg:gsre} batches all currently-ready nodes per step. The batch composition is immaterial to correctness, as we will show.

\begin{lemma}[Local input determinism]
\label{lem:local-input}
At line~14 of \Cref{alg:gsre}, the input to $\NodeProc_\theta$ for any $n \in \mathcal{B}$ consists exclusively of $\tau_n$, $\Spec_n$, and the resolved values of $\Parents_n$. It does not depend on the identity, type, depth, or value of any other node in the graph.
\end{lemma}

\begin{proof}
By inspection of line~14.
\end{proof}

\begin{lemma}[Argument-availability invariant]
\label{lem:arg-avail}
Throughout the execution of \Cref{alg:gsre}, whenever $n$ enters $\EXEC$ every $p \in \Parents_n$ satisfies $\NState(p) = \RESOLVED$.
\end{lemma}

\begin{proof}
Transitions into $\EXEC$ occur only at line~13, reached only for $n \in \mathcal{B} \subseteq R$. The set $R$ was constructed at line~4 to consist precisely of $\PENDING$ nodes whose parents are all $\RESOLVED$. Once a node is $\RESOLVED$ its state is monotone, so the predicate continues to hold at line~13.
\end{proof}

\subsection{Operational properties}

\begin{theorem}[Determinism]
\label{thm:gsre-determinism}
Let $\NodeProc_\theta$ be a deterministic function of its input. Then for any well-formed notion DAG $G$ and any leaf assignment $\sigma$, the value assignment returned by \Cref{alg:gsre} is uniquely determined by $G$, $\sigma$, and $\theta$.
\end{theorem}

\begin{proof}
We show that $v_n$ is uniquely determined for each $n$ by induction on $\dep(n)$. The base case is line~2, which assigns $v_n = \sigma(n)$ to every leaf. For the inductive step, suppose every $p \in \Parents_n$ has its value uniquely determined; \Cref{thm:gsre-termination} below ensures that $n$ eventually enters $R$. At line~14, the input to $\NodeProc_\theta$ at $n$ is $(\tau_n, \Spec_n, (v_p)_{p \in \Parents_n})$ (\Cref{lem:local-input}), each component uniquely determined by $G$, $\sigma$, $\theta$, and the inductive hypothesis. Determinism of $\NodeProc_\theta$ fixes $v_n$.

The batch composition $\mathcal{B}$ does not affect any individual node's input (\Cref{lem:local-input}), so the same value is produced regardless of how ready nodes are batched.
\end{proof}

\begin{theorem}[Order independence]
\label{thm:gsre-order}
Under the hypothesis of \Cref{thm:gsre-determinism}, the value assignment produced by \Cref{alg:gsre} is invariant under any choice of $\mathcal{B} \subseteq R$ at each iteration, provided $\mathcal{B} \ne \emptyset$ whenever $R \ne \emptyset$.
\end{theorem}

\begin{proof}
By \Cref{lem:local-input}, the input presented to $\NodeProc_\theta$ at any node $n$ depends only on $\tau_n$, $\Spec_n$, and parent values---not on other members of the dispatched batch. The set of nodes that ever reach $\RESOLVED$ is determined by the predicate of line~4 plus the absence-of-failure condition; under a deterministic processor that does not return $\bot$, every non-leaf node eventually has all parents resolved by induction on depth, and the output values match across schedules by \Cref{thm:gsre-determinism}.
\end{proof}

The architecture commits only to dependency-respecting evaluation. Among all schedules that respect dependencies, the executor's output is invariant.

\begin{theorem}[Termination]
\label{thm:gsre-termination}
\Cref{alg:gsre} terminates in at most $|V|$ iterations of the main loop, with the number of $\NodeProc_\theta$ invocations bounded by $|V| - |V_{\mathrm{leaf}}|$.
\end{theorem}

\begin{proof}
At each iteration with $R \ne \emptyset$, at least one node transitions from $\PENDING$ to $\READY$ at line~5, and subsequently to $\RESOLVED$ or $\FAILED$. We claim $R \ne \emptyset$ whenever the loop has not yet returned. The set of $\PENDING$ nodes forms a sub-DAG of $G$; among the $\PENDING$ nodes, those whose parents lie outside the $\PENDING$ set form a nonempty set provided the $\PENDING$ set is nonempty and the parent relation is acyclic. These nodes are precisely $R$. The loop therefore terminates after at most $|V|$ iterations, with each non-leaf node invoking the processor at most once.
\end{proof}

\begin{theorem}[Soundness against the oracle]
\label{thm:gsre-soundness}
Let $\NodeProc_\theta$ be deterministic and primitive-correct: for every primitive $\tau$ and every typed input tuple $(v_1, \dots, v_{k_\tau})$, the output of $\NodeProc_\theta$ on input $(\tau, \cdot, (v_1, \dots, v_{k_\tau}))$ equals $f_\tau(v_1, \dots, v_{k_\tau})$. Then for any reasoning task $(G, \sigma)$, the value $v_r$ returned by \Cref{alg:gsre} equals the oracle ground-truth answer.
\end{theorem}

\begin{proof}
Induction on $\dep(n)$. For $\dep(n) = 0$, $v_n = \sigma(n) = \hat{v}_n$ from line~2. For $\dep(n) > 0$, by the inductive hypothesis $v_p = \hat{v}_p$ for all $p \in \Parents_n$. Primitive correctness of $\NodeProc_\theta$ yields $v_n = f_{\tau_n}(v_{p_1}, \dots, v_{p_{k_{\tau_n}}}) = f_{\tau_n}(\hat{v}_{p_1}, \dots, \hat{v}_{p_{k_{\tau_n}}}) = \hat{v}_n$. Applying to $n = r$ completes the proof.
\end{proof}

\Cref{thm:gsre-soundness} isolates the only source of error in the executor: the only way the executor can deviate from oracle evaluation is if the processor is wrong on some primitive in the graph. This isolation is what makes the depth-decoupling analysis in \Cref{sec:information} possible.

\subsection{The node processor as a learning target}

The node processor $\NodeProc_\theta$ is a small Transformer. Its input at a node $n$ consists of a type embedding for $\tau_n$, an encoding of $\Spec_n$, and the resolved values of $\Parents_n$ formatted with type-aware delimiters. Its output is a value of type $\mathsf{out}_{\tau_n}$ produced through a type-appropriate head: regression for continuous types, classification for finite types, copy mechanisms for variable-length structured types.

The processor has no access to a global graph embedding, no access to the depth or index of the node it is processing, and no access to a history vector summarising prior nodes. We refer to the tuple $(\tau_n, \Spec_n, (v_p)_{p \in \Parents_n})$ as the \emph{local context} of $n$, and this is precisely what the processor sees. The consequences of this design for generalisation are developed in \Cref{sec:generalization}.

\section{Architectural Comparison}
\label{sec:arch-comparison}

The executor's inductive bias is best understood by contrast with two natural alternatives: a weight-tied graph transformer with a causal ancestor mask, and a sequence transformer over the serialised DAG. Both alternatives have access to identical input information; they differ in how they process it.

\subsection{Weight-tied graph transformer with causal ancestor masking}
\label{sec:b4tied}

\begin{definition}[Weight-tied graph transformer]
\label{def:b4tied}
Fix a topological ordering $\pi : V \to \{1, \dots, N\}$ of $G$. The weight-tied graph transformer with causal ancestor masking computes
\begin{align*}
h^{(0)}_n &= \mathsf{Embed}(\tau_n, \Spec_n) \quad \text{for each } n \in V, \\
h^{(t+1)}_n &= \mathsf{Layer}_\theta\!\Bigl(h^{(t)}_n;\; \{h^{(t)}_{n'} : n' \in \mathsf{Ancestors}_\pi(n)\}\Bigr), \quad t = 0, \dots, T-1, \\
v_n &= \mathsf{Head}_{\mathsf{out}_{\tau_n}}\!\bigl(h^{(T)}_n\bigr),
\end{align*}
where $\mathsf{Ancestors}_\pi(n) = \{n' : \pi(n') < \pi(n)\}$, $\mathsf{Layer}_\theta$ is a self-attention block restricted by an ancestor mask with parameters shared across $t$, and $\mathsf{Head}_\sigma$ is a type-specific output head.
\end{definition}

The defining features of this architecture are (i) per-layer parameters $\theta$ are shared across $t$, mirroring the executor's reuse of the processor; (ii) attention is restricted to topological ancestors, preserving the causal structure of the DAG; (iii) all node values are computed jointly across $T$ rounds rather than node by node in dependency order.

Property (iii) is the substantive difference from the executor. A node's representation $h^{(t)}_n$ at layer $t > 0$ depends on layer-$t$ representations of ancestors, not on the resolved values of parents. The architecture has $T$ rounds in which to propagate information; nothing enforces that ancestors are fully resolved before descendants are updated.

\subsection{Sequence transformer over the serialised DAG}
\label{sec:b5}

\begin{definition}[Sequence transformer over the serialised DAG]
\label{def:b5}
Let $\mathsf{Serialize}(G)$ be a canonical token sequence encoding of $G$: for each node in topological order, emit tokens for its identifier, type, spec, and parent identifiers, separated by delimiters. The sequence transformer baseline passes $\mathsf{Serialize}(G)$ through a standard sequence transformer (decoder-only or encoder--decoder) and produces values via type-specific heads at the positions corresponding to each node.
\end{definition}

The sequence transformer has the same information as the other two architectures but presented as a flat token sequence with no explicit graph structure beyond what positional embeddings and learned attention can recover.

\subsection{Expressivity containment}

The executor is a strictly more constrained function class than the weight-tied graph transformer.

\begin{proposition}[Expressivity containment]
\label{prop:containment}
For every choice of executor parameters $\theta_{\GSRE}$ there exist weight-tied graph transformer parameters $\theta_{\mathrm{GT}}$ and a depth $T \ge \dep(G)$ such that for every well-formed notion DAG $G$ with $\dep(G) \le T$, the value assignment produced by the executor equals the value assignment produced by the graph transformer.
\end{proposition}

\begin{proof}
Set $T \ge \dep(G)$. Construct the layer so that at layer $t$, a node $n$ with $\dep(n) = t$ attends to its parents (already updated to their final values at layer $t$ by the inductive construction), copies their values into its working state, and applies $\NodeProc_{\theta_{\GSRE}}$ to compute its own value. For all $t' > t$, the node's representation passes through unchanged. The ancestor mask permits this attention pattern because parents are ancestors.

Conversely, the graph transformer has degrees of freedom the executor does not: nothing prevents a layer from blending information from non-parent ancestors, nor from updating a node based on layer-$t$ values of nodes whose semantic values have not yet been computed. These functions are realisable by the graph transformer but not by the executor.
\end{proof}

The containment is what we are after. The executor's smaller hypothesis class encodes the prior that computation respects dependencies; the graph transformer must learn this prior from data.

\section{Information-Theoretic Properties}
\label{sec:information}

We now characterise two properties of the executor: its per-step information budget is independent of graph depth, and under a stochastic processor with bounded per-invocation error, the executor's success probability decays geometrically in depth with no growth in parameters.

\subsection{Per-step information budget}

Let $\mathsf{Enc} : \Val \to \{0,1\}^\ast$ be a fixed bit encoding of values, and write $|\mathsf{Enc}(v)|$ for the length of the encoding of $v$. The primitive description length of a node $n$ is
\[
\ell(n) \;=\; |\mathsf{Enc}(\tau_n)| + |\mathsf{Enc}(\Spec_n)| + \sum_{p \in \Parents_n} |\mathsf{Enc}(v_p)|.
\]

\begin{proposition}[Per-step bounded input]
\label{prop:per-step-bounded}
The input length presented to the processor at any node $n$ is exactly $\ell(n)$, and
\[
\ell(n) \;\le\; |\mathsf{Enc}(\tau_n)| + |\mathsf{Enc}(\Spec_n)| + k_{\tau_n} \cdot L_{\max},
\]
where $L_{\max} = \max_{v \in \Val} |\mathsf{Enc}(v)|$. If $L_{\max}$ and $k_{\max} := \max_{\tau \in \Lib} k_\tau$ are finite, the per-step input length is $\mathcal{O}(k_{\max} L_{\max})$, independent of $|V|$ and $\dep(G)$.
\end{proposition}

\begin{proof}
\Cref{lem:local-input} gives the exact input enumeration. Bounding each parent value by $L_{\max}$ yields the second inequality.
\end{proof}

The sequence transformer's per-step input is $\Theta(|V| \cdot L_{\max})$; the weight-tied graph transformer's per-layer attention scope is $\Omega(d)$ in a chain of depth $d$. The executor is the only one of the three architectures with a per-step input length independent of graph size.

\subsection{Capacity--depth decoupling}

\begin{theorem}[Capacity--depth decoupling under i.i.d.\ primitive error]
\label{thm:capacity-depth}
Let $\NodeProc_\theta$ be a stochastic processor with independent error events at distinct invocations, and let $\eps$ be a uniform upper bound on the probability that $\NodeProc_\theta$ returns a value other than $f_\tau$ on a single primitive-correct input. For any well-formed notion DAG $G = (V, r)$,
\[
\Prob\bigl[v_r = \hat{v}_r\bigr] \;\ge\; (1-\eps)^{|V| - |V_{\mathrm{leaf}}|}.
\]
For chain-shaped DAGs of depth $d$ with no shared subexpressions, this specialises to $\Prob[v_r = \hat{v}_r] \ge (1-\eps)^d$, with the parameter count of $\NodeProc_\theta$ constant in $d$.
\end{theorem}

\begin{proof}
Conditional on every non-leaf node having received its correct parent values, each invocation independently succeeds with probability at least $1 - \eps$. The joint success of all $|V| - |V_{\mathrm{leaf}}|$ non-leaf invocations therefore has probability at least $(1-\eps)^{|V| - |V_{\mathrm{leaf}}|}$, and this event implies $v_r = \hat{v}_r$. The parameter count of $\NodeProc_\theta$ is fixed by the architecture and does not depend on $|V|$ or $d$.
\end{proof}

The bound is loose where errors at downstream nodes can absorb upstream errors (e.g., $\mathsf{ADD\_MOD}$ followed by a $\mathsf{COMPARE}$ that ignores the perturbed bit). It is tight where each primitive's output is fully determined by its parent values and the error events factor.

\begin{remark}[Failure mode of the sequence baseline]
\label{rem:b5-no-decoupling}
A sequence transformer trained on length-$L_{\mathrm{train}}$ serialised DAGs and tested on length-$L_{\mathrm{test}}$ DAGs faces (i) positional extrapolation~\cite{press2021train}, (ii) attention dispersion across longer sequences~\cite{anil2022exploring}, and (iii) loss of independence between primitive evaluations, since all node values are produced by a single coupled latent computation. There is no analogue of \Cref{thm:capacity-depth} in this setting.
\end{remark}

\begin{remark}[Failure mode of the graph transformer baseline]
\label{rem:b4tied-conditional-decoupling}
The weight-tied graph transformer has $T$ layers of shared parameters; depth-$d$ graphs require $T \ge d$ for information to propagate from leaves to root. Even when $T$ is sufficient, per-layer attention is over $\Omega(d)$ ancestors in a chain, which couples primitive evaluations through softmax temperature and attention dispersion. The independence factorisation of \Cref{thm:capacity-depth} does not hold.
\end{remark}

\section{Generalisation Properties}
\label{sec:generalization}

\subsection{Local-context invariance}

\begin{definition}[Local context]
\label{def:local-context}
The local context of a node $n$ is the tuple
\[
\mathsf{Ctx}(n) \;=\; \bigl(\tau_n,\; \Spec_n,\; (v_p)_{p \in \Parents_n}\bigr).
\]
\end{definition}

\begin{proposition}[Locality of the processor]
\label{prop:gsre-local-only}
For any two well-formed DAGs $G, G'$ and any nodes $n \in V_G$, $n' \in V_{G'}$ with $\mathsf{Ctx}(n) = \mathsf{Ctx}(n')$, the value produced by the executor at $n$ equals the value produced at $n'$.
\end{proposition}

\begin{proof}
Immediate from \Cref{lem:local-input} and determinism of $\NodeProc_\theta$.
\end{proof}

The executor sees only local contexts. Aggregated quantities of the graph---size, depth, ancestor topology, sibling identity---never reach the processor.

\subsection{Marginal invariance and depth generalisation}

Let $\mathcal{D}_d$ denote a distribution over notion DAGs of depth $d$ paired with leaf assignments. Let $\mathcal{M}_d$ denote the marginal distribution of local contexts induced by drawing $G \sim \mathcal{D}_d$ and then drawing an internal node uniformly.

\begin{theorem}[Depth-independent per-node error under marginal invariance]
\label{thm:local-context-ood}
If $\mathcal{M}_{d_1} = \mathcal{M}_{d_2}$ then the expected per-node error rate of the executor's processor under $\mathcal{D}_{d_1}$ equals that under $\mathcal{D}_{d_2}$.
\end{theorem}

\begin{proof}
The per-node error of the processor is a function $\phi : \mathsf{Ctx} \to [0,1]$ of the local context alone. The expected per-node error under $\mathcal{D}_d$ equals $\int \phi \, d\mathcal{M}_d$. Equality of the marginals gives equality of the integrals.
\end{proof}

Combining the two:

\begin{corollary}[Depth generalisation under marginal invariance]
\label{cor:ood-accuracy}
Let $\mathcal{D}_{d_{\mathrm{train}}}$ and $\mathcal{D}_{d_{\mathrm{test}}}$ have equal local-context marginals. Let $\eps$ be the per-node processor error rate under $\mathcal{D}_{d_{\mathrm{train}}}$. Then the executor satisfies
\[
\mathrm{Acc}_{\mathcal{D}_{d_{\mathrm{test}}}}(\GSRE) \;\ge\; \E_{G \sim \mathcal{D}_{d_{\mathrm{test}}}}\Bigl[(1-\eps)^{|V_G| - |V_{G,\mathrm{leaf}}|}\Bigr].
\]
\end{corollary}

The right-hand side depends on $d_{\mathrm{test}}$ only through the size distribution of the test graphs, and decays geometrically rather than catastrophically.

The marginal-invariance assumption is non-trivial. It is approximately satisfied when leaf values are sampled uniformly, primitive choices induce uniform output distributions, and the primitive library is closed under composition in a way that preserves the marginal distribution of internal-node parent values. Modular arithmetic over a prime modulus is a clean special case: for $p$ prime, $(\mathbb{Z}/p\mathbb{Z})^{\times}$ is cyclic of order $p - 1$, and the induced output distributions of $\mathsf{ADD\_MOD}, \mathsf{SUB\_MOD}, \mathsf{MUL\_MOD}$ on independent uniform inputs remain uniform on $\mathbb{Z}/p\mathbb{Z}$ (or on $(\mathbb{Z}/p\mathbb{Z})^{\times}$ for the multiplicative case, conditioned on nonzero inputs). In such settings $\mathcal{M}_d$ is approximately invariant in $d$, and \Cref{cor:ood-accuracy} applies directly.

\subsection{Failure modes of the baselines under depth shift}

For the weight-tied graph transformer, the number of ancestors of a node grows with depth. Attention aggregates over this set, and the marginal distribution of attention input sets under $\mathcal{D}_d$ shifts with $d$. There is no fixed-size per-step input, so the local-context invariance argument breaks down.

For the sequence transformer, the serialised DAG length grows with $|V|$, which grows with depth. Position embeddings, attention dispersion, and softmax temperature interact with sequence length, and the marginal distribution of inputs to the network shifts with depth.

The executor avoids both failure modes by construction: the per-step input is bounded, and only local contexts ever reach the processor.

\section{Learned Graph Construction}
\label{sec:learned-construction}

The executor as developed so far assumes a pre-specified notion DAG. The architecture extends to the case where the DAG is constructed on the fly by a learned policy over a fixed primitive library. We formalise this construction as a Markov decision process with deterministic transitions.

\subsection{The graph-construction MDP}

\begin{definition}[Graph-construction MDP]
\label{def:gcmdp}
The graph-construction MDP for a task specification $q$ over library $\Lib$ is the tuple $(\mathcal{S}, \mathcal{A}, P, R, \gamma)$:

\textbf{State.} An element of $\mathcal{S}$ is a pair $(G, q)$ where $G$ is a partial well-formed notion DAG over $\Lib$ together with a partial value assignment $v$. The initial state is $s_0 = (\emptyset, q)$.

\textbf{Action.} An action is a triple $a = (n_{\mathrm{select}}, \tau, b)$:
\begin{itemize}[topsep=2pt,itemsep=2pt]
  \item $n_{\mathrm{select}}$ is either a $\PENDING$ node in $G$ with an unfilled slot, or a fresh node identifier.
  \item $\tau \in \Lib$ is a primitive whose output type matches the demanded type at $n_{\mathrm{select}}$.
  \item $b$ binds each input slot of $\tau$ either to an existing resolved value in $G$'s value pool, or to a new sub-goal of the demanded type.
\end{itemize}
A distinguished action $\texttt{FAIL}$ terminates the episode.

\textbf{Transition.} $P((G, q), a)$ deterministically produces $G'$ by attaching $\tau$ at $n_{\mathrm{select}}$ with bindings $b$, then runs the traversal engine to resolve any newly-ready nodes.

\textbf{Reward.} $R(s, a, s') = +1$ if $s'$ contains a fully resolved root whose value equals the task ground truth; otherwise $R = 0$.

\textbf{Discount.} $\gamma \in (0, 1)$.
\end{definition}

A policy $\pi : \mathcal{S} \to \Delta(\mathcal{A})$ defines a distribution over actions. The Graph Policy Network $\GPN_\phi$ parameterises such a policy.

\subsection{Action space cardinality}

\begin{proposition}[Per-step action space]
\label{prop:action-cardinality}
At a state with $m$ unfilled slots across all $\PENDING$ nodes and $v$ resolved values in the value pool,
\[
|\mathcal{A}(s)| \;\le\; (m + 1) \cdot |\Lib| \cdot (v + |\Lib|)^{k_{\max}} + 1.
\]
\end{proposition}

\begin{proof}
$m + 1$ choices of expansion target (existing slot or new root sub-goal); $|\Lib|$ choices of primitive; for each of up to $k_{\max}$ input slots, a binding to one of $v$ resolved values or $|\Lib|$ new sub-goals; one extra action for $\texttt{FAIL}$.
\end{proof}

For a library of size $|\Lib| = K$, maximum arity $k_{\max} = 2$, and a value pool of modest size, the action space is polynomial in the partial-graph state and tractable for standard policy-gradient methods.

\subsection{Deterministic transitions and search}

The transition function of \Cref{def:gcmdp} is deterministic conditional on the seed of $\NodeProc_\theta$. This permits Monte Carlo tree search over expansion sequences with the policy network as prior and a value head as leaf evaluator, in the style of AlphaZero~\cite{silver2018general}. Tree search interacts cleanly with the executor because every rollout is itself a deterministic partial execution of the underlying \GSRE.

\subsection{Training signal}

The reward is sparse: $+1$ only when a fully-resolved root matches the ground truth. Two standard mechanisms apply.

\paragraph{Behaviour cloning bootstrap.} When ground-truth expansion traces are available (from oracle DAG generation at low depth), the policy can be initialised by supervised loss
\[
\mathcal{L}_{\mathrm{BC}}(\pi_\phi) \;=\; -\E_{(s, a^\ast)}\bigl[\log \pi_\phi(a^\ast \mid s)\bigr],
\]
where $a^\ast$ is the oracle expansion action at $s$.

\paragraph{Policy-gradient refinement.} Subsequent training proceeds via PPO~\cite{schulman2017proximal} or analogous policy-gradient updates on the sparse-reward signal, possibly augmented with MCTS-improved policy targets~\cite{silver2018general}. Because transitions are deterministic, value estimates depend only on the policy itself and the (fixed-seed) executor.

\section{Library Extension}
\label{sec:library-extension}

The most ambitious extension of the architecture is the extension of the primitive library itself: identifying reusable subgraphs in successful traces, abstracting them into new primitives, and incorporating these into the library. The framework admits a clean formal statement of the problem.

\subsection{Compositional compression}

Let $\mathcal{T} = \{G_1, G_2, \dots\}$ be a collection of successfully-resolved notion DAGs. A subgraph abstraction is a typed parameterised graph template $T$ together with embeddings $\iota_i : T \hookrightarrow G_i$ for each $G_i$ in some $\mathcal{T}_T \subseteq \mathcal{T}$, where each $\iota_i$ preserves the structure of $T$ up to substitution of free type variables and leaves.

\begin{definition}[Compositional compression value]
\label{def:compression}
The compression value of an abstraction $T$ over $\mathcal{T}$ is
\[
\Delta(T) \;=\; \sum_{i \,:\, T \hookrightarrow G_i} (\mathsf{count}_T(G_i) - 1) \cdot |T| \;-\; |T|,
\]
where $\mathsf{count}_T(G_i)$ is the number of structurally-distinct embeddings of $T$ in $G_i$, and $|T|$ is the description length of $T$.
\end{definition}

The library-learning problem is to identify $T$ with large $\Delta(T)$. It generalises subgraph-mining problems studied in the graph-data-mining literature~\cite{washio2003state} and connects directly to library learning in inductive program synthesis~\cite{ellis2021dreamcoder}. The novel features in the present setting are the typed parameterisation of abstractions and the requirement that abstractions remain executable by the shared processor.

\subsection{Continual learning of the processor}

Adding a new primitive $\tau_{K+1}$ to the library requires the processor to learn the new operation. Two constraints conflict:

\begin{itemize}[topsep=2pt,itemsep=2pt]
  \item \emph{Stability.} Per-primitive accuracy on $\{\tau_1, \dots, \tau_K\}$ must not degrade after training on $\tau_{K+1}$.
  \item \emph{Plasticity.} Time to reach $1 - \eps$ accuracy on $\tau_{K+1}$ should be sublinear in $K$.
\end{itemize}

Standard tools---adapter layers~\cite{hu2021lora}, elastic-weight consolidation~\cite{kirkpatrick2017overcoming}, mixture-of-experts---trade these properties against each other. The shared-processor architecture commits to a single set of parameters for all primitives; reconciling stability and plasticity in this constrained setting is open.

\subsection{Verification of invented primitives}

A new primitive $\tau_{K+1}$ induced by abstraction must be verified to compute a useful function. Two interpretations of usefulness are operationally distinct.

\begin{itemize}[topsep=2pt,itemsep=2pt]
  \item \emph{Compression-based.} $\tau_{K+1}$ is admitted to the library iff it reduces the description length of solutions on a held-out problem set by an amount exceeding the cost of its parameters.
  \item \emph{Semantic.} $\tau_{K+1}$ must compute a well-defined function. Absent an external oracle, this function is whatever the processor produces, and correctness is a property of the abstraction process itself.
\end{itemize}

Compression-based verification is realisable and is the criterion used by DreamCoder. Semantic verification raises the foundational question of whether the system can construct test cases for its own invented primitives, which in the present setting reduces to whether the example traces from which $\tau_{K+1}$ was mined constitute a sufficient defining set.

\subsection{Search versus abstraction}

A library-extending agent must decide, for any novel problem, whether to search the existing library or to introduce a new primitive. The two options have distinct costs: search scales with the size of the action space at the relevant depth; abstraction scales with the size of the trace corpus.

\section{Connections to Prior Work}
\label{sec:related}

The architecture connects to several established lines of work.

\paragraph{Compositional generalisation.} The SCAN benchmark~\cite{lake2018generalization} and its successors~\cite{kim2020cogs, hupkes2020compositionality} established the empirical phenomenon that motivates this architecture. Subsequent work demonstrated architectural and data-side mitigations~\cite{russin2019compositional, csordas2021devil, andreas2020good}. The present architecture mitigates the gap by structural enforcement.

\paragraph{Chain-of-thought and scratchpad reasoning.} Wei et al.~\cite{wei2022chain} and Nye et al.~\cite{nye2021show} expose intermediate reasoning steps as targets in token form. The notion DAG is the structural counterpart: the same intermediate steps, represented explicitly rather than implicitly in a token sequence.

\paragraph{Universal and recurrent transformers.} Weight-tied transformers~\cite{dehghani2018universal} are the sequence analogue of the weight-tied graph transformer of \Cref{sec:b4tied}. Length-generalisation behaviour of recurrent architectures has been investigated by~\cite{schwarzschild2021can}.

\paragraph{Neural module networks.} Andreas et al.~\cite{andreas2016neural} construct computation graphs from a fixed module library by parsing the input question. The notion DAG generalises this to an arbitrary typed DAG with a shared module.

\paragraph{Program synthesis and library learning.} The learned-graph-construction MDP of \Cref{sec:learned-construction} is a neural program synthesis problem with a known DSL~\cite{gulwani2017program, devlin2017robustfill}. The library-extension setting of \Cref{sec:library-extension} is most closely related to DreamCoder~\cite{ellis2021dreamcoder}.

\paragraph{Graph neural networks.} The weight-tied graph transformer is structurally close to message-passing GNNs~\cite{kipf2016semi, velivckovic2017graph}, differing in that GNN message passing updates all nodes at every layer regardless of whether their inputs have stabilised.

\section{Discussion}
\label{sec:discussion}

The executor's key inductive bias is the hard enforcement of dependency order. Among architectures with access to identical information, only the executor commits to the constraint that a node's value is computed once, after all of its inputs are available, by a function whose input is restricted to the node's local context. Weight-tied graph transformers approximate this constraint via causal ancestor masking; sequence transformers do not approximate it at all.

The consequence is a clean separation of concerns. The neural component is responsible only for executing primitives; the symbolic component is responsible for tracking dependencies and scheduling. Each can be analysed independently. The neural component's error is bounded by primitive-level training; the symbolic component's correctness follows by structural induction on the DAG.

The capacity--depth decoupling of \Cref{thm:capacity-depth} is the architectural payoff. A small processor trained to high primitive accuracy can execute arbitrarily deep graphs, with success probability decaying geometrically rather than catastrophically and with no growth in parameters. The local-context invariance of \Cref{prop:gsre-local-only} ensures that this decoupling extends to graphs of depths not seen in training, provided the marginal distribution of local contexts is preserved.

The extensions to learned graph construction and library extension preserve this structure. The construction policy is a separate neural component that decides what graph to build; the executor as developed here resolves whatever graph it is given. The library-extension problem becomes a problem of mining and verifying new abstractions, with the executor's structural guarantees serving as ground for the verification step.
